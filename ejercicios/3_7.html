Problema 3.7

Clasificador de Ventas

Escribe un programa que procese una serie de ventas. Primero, debe preguntar cuántas ventas se van a introducir. Luego, pedirá el monto de cada una y, al final, mostrará un resumen de cuántas ventas fueron chicas, medianas y grandes.

Requisitos

  • Define tres contadores: ventasChicas, ventasMedianas, ventasGrandes.
  • Pide al usuario la cantidad total de ventas a procesar.
  • Usa un bucle for que se repita esa cantidad de veces.
  • Dentro del bucle, pide el monto de cada venta.
  • Clasifica cada monto y aumenta el contador correspondiente:
    • Chicas: si el monto es menor o igual a 200.
    • Medianas: si es mayor a 200 pero menor a 400.
    • Grandes: si es mayor or igual a 400.
  • Al final del bucle, imprime el total de cada categoría.

Ejemplo de uso esperado

Introduce la cantidad de ventas a procesar: 5
Introduce el monto de la venta #1: 150
Introduce el monto de la venta #2: 300.50
Introduce el monto de la venta #3: 400
Introduce el monto de la venta #4: 199.99
Introduce el monto de la venta #5: 550
Resumen de ventas:
Ventas chicas (<= 200): 2
Ventas medianas (< 400): 1
Ventas grandes (>= 400): 2

Concepto Clave: Bucles

Este ejercicio introduce los bucles (o ciclos). Un bucle for es perfecto para repetir una acción un número conocido de veces, como pedir el monto de cada venta.

Tip: Operador de Incremento

Dentro del bucle, para aumentar un contador, puedes usar el operador de incremento ++.

Escribir ventasChicas++; es un atajo muy común para ventasChicas = ventasChicas + 1;. ¡Es más corto y fácil de leer!
